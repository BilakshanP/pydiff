## What this tool is

A Python 3 CLI (`pydiff`) that generates a self-contained HTML report comparing git refs. Pure stdlib, no third-party deps. Installable from git. README.md describes user-facing features.

## Architecture

1. Resolve each ref via `git rev-parse --verify`. The sentinel `.` (worktree) is short-circuited: `resolve()` returns `"."` unchanged without calling git.
2. Classify ref as branch / remote-branch / tag / commit / worktree via `git rev-parse --symbolic-full-name` (with special-cases: `.` → worktree; `HEAD` and anything containing `~^@:` → commit).
3. Enumerate changes: `git diff --name-status -M -z <base> <target>` → list of `(status, old_path, new_path)`. When target is worktree (`.`), the second ref is dropped so git compares against the working tree. When `--untracked` is set on a worktree target, `git ls-files --others --exclude-standard -z` is merged in as `A` entries.
4. Fetch blobs: `git show <ref>:<path>` for refs; for the worktree sentinel, read `os.path.join(repo, path)` directly. Binary detection via `\0` in first 8KB (bytes on disk, str for git show). Missing-file / UnicodeDecodeError return sentinel strings.
5. Generate diff HTML: `difflib.HtmlDiff().make_table(from_lines, to_lines, ...)`.
6. **Post-process** the single table into two independent tables (`split_diff_table`):
   - `HtmlDiff` emits 6 cells per row: `[L-next, L-lineno, L-code, R-next, R-lineno, R-code]`.
   - Drop the "next" nav columns (they break once split).
   - Keep `[lineno, code]` per side.
   - Inject `&nbsp;` into empty code cells to keep row heights identical across panes.
   - Single-cell rows (hunk separators) are duplicated as-is with `colspan=2`.
7. Render: DOCTYPE + inline CSS + body + inline `<script>` + single HTML file. CSS and JS are loaded at `html_render` import time from the `pydiff.assets` package via `importlib.resources`, then wrapped in `<style>` / `<script>` tags.

## Scroll model (critical — don't break)

- **Sync mode (default)**: outer `.split` is the single vertical scroller (`max-height: 70vh; overflow-y: auto`). `.pane-inner` handles horizontal scroll per pane. Native smooth.
- **Nosync mode** (`.split.nosync`): `.split` has no max-height. Each `.pane` gets its own `overflow: auto; max-height: 70vh`. `.pane-inner` becomes non-scrolling (horizontal scroll on pane directly).
- Toggle preserves scroll position via `requestAnimationFrame` handoff in JS.
- Handle uses `position: sticky; top: 0; align-self: stretch` — stays pinned to visible viewport in sync mode, becomes a normal flex child in nosync.

**Do NOT reintroduce JS-based scrollTop syncing** — it caused jerkiness the user specifically complained about. The current CSS-only approach is what they like.

## JS features

All scoped to `.file-content` parent (not to `<summary>` — buttons inside `<summary>` conflict with native toggle; controls are in a sibling `.controls` div):

- **View toggle** (data-view=both|left|right): adds/removes `.hidden`/`.solo` classes on panes + `.hidden` on handle.
- **Sync toggle** (data-sync-toggle): toggles `.nosync` class on `.split`, updates button label "Sync: on"/"Sync: off", transfers scroll position.
- **Drag handle**: mousedown/move/up, clamped 10–90%, only active when `data-view === 'both'`. Sets `flex-basis` on both panes.
- **Fullscreen toggle** (data-expand): toggles `.fullscreen` on closest `.file-container` or `.target-section`. Only one at a time. Esc to exit. Icon swaps `⛶` ↔ `✕`.
- **Target prev/next FAB**: shown only when >1 targets; finds the current target by scroll position + offset, jumps to adjacent.

## Known behaviours / gotchas

- **Rename with heavy edits** may fall below git's similarity threshold (default ~50%) and appear as delete + add. Expected.
- **HEAD classification**: Git resolves `HEAD` to `refs/heads/<current>`. Special-case in `classify()` treats HEAD and any ref containing `~^@:` as commit (not branch).
- **Binary detection**: just `\0` in first 8KB of text. Not bulletproof; real binary files that git didn't already filter would show as `<Binary or non-UTF-8 file>`.
- **Empty cell row-height fix**: the `&nbsp;` injection in `split_diff_table` is load-bearing — without it, pure add/delete rows render at different heights on left vs right, and sync scroll drifts.
- **anchor_id** slugifies non-alphanumeric chars to `-`, prefixed with `f-`. Collision-prone for files that differ only in punctuation but fine in practice.
- **`subprocess.run(..., check=True, capture_output=True, text=True)`**: note `text=True` is needed because we treat output as str.
- **Tests use `demo_repo/` as a fixture.** Tests assert the fixture is clean at startup (`test_demo_repo_is_clean`); the `WorktreeMutation` helper snapshots and restores file contents around each mutation. A failing cleanup would leak into later runs — if tests start failing mysteriously, check `git -C demo_repo status`.

## Conventions to preserve under basedpyright / ruff

- Extract `argparse.Namespace` fields via `typing.cast` at the top of `render()` — don't reference `args.X` throughout the function, or `reportAny` comes back.
- Prefix unused returns with `_ =` (applies to `argparse.add_argument`, `f.write`, `list.pop`, etc.).
- Avoid adjacent-literal string concatenation inside `append(...)` — use explicit `+` between f-strings.
- Type regex `findall` results explicitly when the tuple shape is known.
- Only use `f"..."` when the string actually interpolates — ruff's `F541` flags bare `f"..."` literals.
- Run `ruff format` before committing; the project follows its default style.
